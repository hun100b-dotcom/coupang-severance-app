## 퇴직금 로직 검증 보고서

작성일: 2026-03-11

### 1. 실데이터 비교

비교 파일:

- `일용근로내역서_고용.pdf`
- `tests/fixtures/ret_cal_coupang.pdf`

실제 퇴직정산서에서 읽힌 핵심 값:

- 퇴직금: `2,485,800원`
- 평균임금 산정 합계: `4,104,176원`
- 평균임금 산정 총일수: `84일`
- 근속기간 관련 수치: `619일`로 읽힘

앱 수정 후 실제 PDF 기준 API 응답:

- 평균임금: `49,335원`
- 근속일수(`work_days`): `616일`
- 인정 근속기간(`qualifying_days`): `616일`
- 퇴직금: `2,497,860원`
- 통상임금 하한선 적용: `false`

차이 분석:

- 기존 앱은 두 자리 날짜(`10`, `22`)가 줄바꿈으로 분리된 PDF 셀을 잘못 파싱하여 최근 3개월 지급액이 과소 집계됐다.
- 기존 앱은 평균임금을 최근 90일 고정 구간(`91일`)로 나눠 실제 정산서보다 더 낮은 일당(`43,272원`)을 만들었다.
- 기존 앱은 퇴직금 계산에 사용하는 일수와 화면에 표시하는 일수가 완전히 일치하지 않았다.
- 수정 후에는 실제 정산서와 거의 같은 수준으로 수렴했다. 남은 차이(`약 12,060원`)는 실제 정산서 PDF OCR상 `4,104,176원`으로 읽힌 값과 앱 파서가 집계한 `4,144,176원`의 차이에서 비롯될 가능성이 높다.

### 2. 적용한 수정 사항

#### PDF 파서

- 파일: `backend/app/services/pdf.py`
- `근로일자` 셀의 개행을 제거해 두 자리 날짜가 잘리지 않도록 수정
  - 예: `9,1\\n0,13` -> `9,10,13`
  - 예: `21,2\\n2,23` -> `21,22,23`

#### 평균임금 산정

- 파일: `backend/app/services/severance.py`
- 최근 3개월 조회창 안에서 실제 첫 근무일 ~ 퇴직일의 총일수(`calendar_days`)를 분모로 사용하도록 수정
- 실데이터 기준:
  - 수정 전: `3,937,748.5 / 91 = 43,271.96원`
  - 수정 후: `4,144,176 / 84 = 49,335.43원`

#### 퇴직금 산정

- 파일: `backend/app/services/severance.py`
- `compute_severance`가 내부적으로 `check_continuous_employment`를 이용해 `qualifying_days`를 사용하도록 수정
- 비대상 케이스(`work_days < 365`)는 퇴직금을 `0원`으로 처리하도록 수정

#### API 응답 정렬

- 파일: `backend/app/api/severance.py`
- `average_wage`, `work_days`, `severance`가 모두 같은 기준(최종 평균임금, 인정 근속기간)으로 내려가도록 정렬
- 평균임금 산정 리포트의 `avg_total_days_in_period`는 실제 근로일수가 아닌 산정 총일수(`calendar_days`)를 사용하도록 수정

#### 프론트 표시

- 파일: `frontend/src/pages/ResultSeverance.tsx`
- 평균임금 리포트의 라벨을 `근무 일수`에서 `산정 일수`로 변경해 의미를 명확히 함
- `work_days`와 `qualifying_days`가 API 응답에서 일치하도록 맞춰 `계속 근로 기간`과 `인정 근속기간`이 모순되지 않도록 정렬

- 이전에 존재하던 `compute_severance`와 API 응답 간 불일치는 해소됐다.

### 4. 남은 리스크

- 실제 정산서 PDF의 OCR 결과상 `4,104,176원`과 앱 파싱 기준 `4,144,176원` 사이에 `40,000원` 차이가 있다.
- 이 값 차이 때문에 실제 정산서 대비 약 `12,060원` 차이가 남아 있다.
- 이는 정산서 PDF OCR 오독 가능성 또는 2021-10 부분기간 급여 산입 방식 차이 때문일 수 있어, 필요하면 해당 월 일급 배분 규칙을 추가 검토해야 한다.

### 5. 결론

- 이번 수정으로 앱의 퇴직금 계산은 실제 정산서에 매우 근접해졌고,
- 화면에 보이는 평균임금, 근속일수, 퇴직금 수식, 최종 금액이 하나의 기준으로 일치하게 됐다.
- 특히 실제 공단 PDF에서 줄바꿈으로 찢긴 날짜 파싱 오류와 평균임금 산정 기간 정의 오류가 핵심 원인이었고, 이를 수정하면서 정합성이 크게 개선됐다.
