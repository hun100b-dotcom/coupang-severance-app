# 퇴직금 한번에 — UI/UX 개선 및 실업급여 통합 계획 보고서

**문서 버전**: 1.0  
**기준일**: 2025년 3월  
**대상**: 쿠팡/물류 일용직 퇴직금·실업급여 확인 Streamlit 앱

---

## 1. 개요

본 보고서는 다음 두 가지를 정리한다.

1. **실업급여 대상 여부 확인** 기능 추가 시, 퇴직금과의 진입 경로(탭/선택 위치)에 대한 권장안 및 타사 BM·UX 분석.
2. **플랜 모드에서 수립한 UI/UX 개선 계획 전체** 요약(인트로 화면, 웜 톤 디자인, 적용 순서).

---

## 2. 실업급여 통합 — 탭 배치 및 BM 권장안

### 2.1 제안된 두 가지 방식

| 방식 | 설명 | 장단점 |
|------|------|--------|
| **A. 통합 인트로 후 선택** | 하나의 인트로 화면 → "퇴직금" / "실업급여" 중 선택 → 해당 플로우 진입 | 장: 첫인상이 통합 서비스로 일관됨. 서비스 가치 전달 후 선택으로 "뭘 할지"가 분명해짐. 단: 인트로가 한 단계 추가됨. |
| **B. 상단 탭 선노출** | 상단에 [퇴직금 \| 실업급여 \| …] 탭 배치 → 탭 클릭 시 해당 서비스 인트로 또는 바로 설문 | 장: 원하는 서비스를 바로 고를 수 있음. 단: 첫 방문자에게 "메뉴"만 보여질 수 있어 가치 전달이 약해지고, 삭막한 포털 느낌이 날 수 있음. |

### 2.2 타사·도구 분석

- **예스폼 자동계산**  
  - 퇴직금·실업급여·4대보험·연봉 등 여러 계산기를 **한 플랫폼**에서 제공.  
  - **상단/좌측에 계산기 목록**을 두고, 사용자가 메뉴에서 "퇴직금계산" 또는 "실업급여"를 선택하면 해당 도구 화면으로 이동.  
  - 특징: "선택 먼저(탭/메뉴)" 구조. 도구가 많아 포털형이 됨.

- **사람인 실업급여 자격 확인**  
  - 실업급여 **한 가지**에 집중한 단일 목적 도구.  
  - 별도 퇴직금 메뉴 없이, 실업급여 자격 확인 플로우만 제공.

- **고용24(노동부)**  
  - 실업급여 간편/상세 모의계산을 **고용형태별**(상용·일용·자영업 등)로 제공.  
  - 퇴직금과는 별도 페이지/메뉴로 분리.

- **토스 앱**  
  - "한 화면, 한 메시지" 원칙.  
  - 최근에는 불필요한 인트로를 줄이고, **가치가 분명할 때만** 첫 화면에서 메시지 전달.

### 2.3 권장 BM·UX: 통합 인트로 + 인트로 내에서 퇴직금/실업급여 선택

**결론: 방식 A(통합 인트로 후 선택)를 권장한다.** 단, "시작하기" 하나가 아니라 **인트로 안에서 퇴직금 / 실업급여 두 가지를 동시에 CTA로 노출**하는 형태가 좋다.

이유 요약:

1. **서비스 가치를 먼저 전달**  
   - "퇴직금이든 실업급여든, 일용직도 한 번에 확인해 보세요"처럼 **한 번의 메인 화면**으로 통합 가치를 보여준 뒤, 사용자가 목적에 맞는 버튼을 누르게 할 수 있음.  
   - 상단 탭만 먼저 보이면 "뭘 하는 서비스지?"가 상대적으로 덜 전달됨.

2. **결정 부담 감소**  
   - 인트로 한 화면에 **큰 이모지 + 짧은 문구 + [퇴직금 확인하기] [실업급여 확인하기]** 두 개의 큰 버튼(카드형 권장)을 두면, "내가 필요한 것 하나만 고르면 된다"는 인지가 쉬움.  
   - 예스폼처럼 메뉴가 많은 포털과 달리, 우리는 퇴직금·실업급여 두 가지로만 초점을 맞출 수 있음.

3. **토스·고객 경험 원칙과 정합**  
   - "한 화면, 한 메시지"에 가깝게, **첫 화면 = "일용직도 퇴직금·실업급여 확인할 수 있어요"** 하나로 통일하고, 그 다음 행동(퇴직금 vs 실업급여)만 두 개로 나누는 구조가 자연스러움.

4. **확장 시**  
   - 나중에 "연차수당" 등 메뉴를 더 추가하더라도, **인트로에서 카드/버튼을 하나 더 추가**하는 방식으로 확장 가능. 상단 탭을 그때 도입해도 늦지 않음.

**권장 플로우 (문장으로)**

- 앱 진입 → **통합 인트로 1장** (큰 이모지 + 서비스 가치 문구 + 보조 문구).  
- 인트로 하단에 **두 개의 동일 위상 CTA**: **[퇴직금 확인하기]** / **[실업급여 확인하기]** (카드형 또는 큰 버튼, 나란히 또는 세로 배치).  
- 클릭 시 `st.session_state.service = "severance" | "unemployment"` 저장 후, 각각 **기존 퇴직금 플로우(Step 1 회사 선택~)** / **실업급여 전용 플로우(자격 질문 → 계산)** 로 진입.  
- 플로우 **내부**에서는, 푸터나 결과 화면 등에 "실업급여도 확인해 보기" / "퇴직금 확인해 보기" 같은 **전환 링크**를 두어, 상단 탭 없이도 다른 서비스로 이동 가능하게 함.

**상단 탭을 두는 경우(선택)**

- 인트로를 거친 **이후** 단계(예: Step 2 이상 또는 결과 화면)에서만 상단에 `[퇴직금 | 실업급여]` 탭을 노출해, 이미 서비스를 이해한 사용자에게만 전환 수단을 제공하는 방식은 가능하다.  
- 단, **첫 화면에서는 탭 없이 통합 인트로 + 두 개 CTA**를 유지하는 것을 우선 권장한다.

## 3. 플랜 모드 계획 전체 요약

아래는 현재 플랜 문서에 정리된 UI/UX 개선 작업을 보고서 형식으로 재구성한 것이다.

### 3.1 목표

- (1) **메인 인트로 화면**으로 서비스 가치 전달 및 "이거 해볼까?" 같은 클릭 유도.  
- (2) **퇴직금 + 실업급여**를 통합 인트로에서 선택 가능하게 구성(위 2.3 반영).  
- (3) **따뜻한 톤**의 디자인(배경·카드·버튼·타이포)으로 삭막함 해소.

---

### 3.2 작업 0: 메인 인트로 화면 추가 (최우선)

- **위치**: `app.py` — `init_session_state`, `main()` 분기, `render_intro()` (및 필요 시 서비스 선택 처리).
- **플로우**  
  - 진입 → **인트로만** 표시 (스텝 인디케이터 없음).  
  - 인트로 구성:  
    - **큰 이모지** (상단 중앙, 4rem~5rem 수준).  
    - **헤드라인**: 서비스 가치 한 줄 (예: "일용직도 퇴직금·실업급여 받을 수 있을까?").  
    - **서브카피**: 2줄 이내 (예: "쿠팡·컬리·CJ 등에서 일하셨다면, 자격 확인부터 계산까지 도와드려요.").  
    - **CTA**: **[퇴직금 확인하기]** / **[실업급여 확인하기]** 두 개 (primary 또는 카드형).  
  - 클릭 시 `intro_done = True` 및 `service = "severance" | "unemployment"` 설정 후 `st.rerun()`, 해당 서비스 Step 1로 진입.
- **스타일**: `.intro-hero` 등 전용 클래스로 이모지·헤드라인·서브·버튼 정렬 및 간격 적용.

---

### 3.3 작업 1~6: 친근한 톤 디자인 (기존 계획)

| 순서 | 내용 |
|------|------|
| 1 | **컬러**: 페이지 배경 `#f7f6f3` 또는 `#faf8f5`, 카드 배경 `#fffefc`/`#fefdfb`, 테두리 웜 그레이(`#e8e6e3` 등), 본문/보조 텍스트 웜 톤, 그림자 소폭 강화. |
| 2 | **헤더·카드**: `.app-header` 상단 포인트 바 또는 연한 그라데이션, `.content-card` 웜 화이트·웜 테두리·그림자. |
| 3 | **버튼**: `.stButton > button` — primary 블루 유지, secondary는 웜 배경·웜 테두리·호버 시 약한 변화. |
| 4 | **스텝 인디케이터·질문**: `.step-dot` 비활성 웜 그레이, `.question-title`/`.question-desc` 웜 텍스트·line-height 1.5~1.6. |
| 5 | **Success/Fail/결과 카드**: Success 연한 그린 틴트, Fail 웜 그레이·공감 문구, `.result-card`/`.chart-card` 웜 배경·테두리. |
| 6 | **기타 위젯**: st.expander, date_input, number_input, text_input, file_uploader 등 웜 배경·테두리·radius 통일. |

### 3.4 적용 순서 (구현 시 권장)

| 순서 | 작업 |
|------|------|
| 0 | 인트로 화면: `intro_done`, `service` 추가, `render_intro()` 구현, 인트로에서 퇴직금/실업급여 선택 시 해당 플로우 진입. |
| 1 | `:root` 및 전역 배경·테두리·텍스트 웜 톤 적용. |

| 3 | `.stButton > button` primary/secondary 전면 스타일링. |
| 4 | `.step-indicator`, `.step-dot`, `.question-*` 색·간격. |
| 5 | `.result-hero.success` / Fail / expander 웜 톤. |
| 6 | expander, input, file_uploader 셀렉터 보조 스타일. |

---

### 3.5 실업급여 플로우(신규)에 대한 기술적 메모

- **세션**: `st.session_state.service` 로 "severance" / "unemployment" 구분.  
- **실업급여 전용**: 자격 요건(예: 이직 전 18개월 중 180일 이상 가입, 비자발적 퇴사 등)에 맞는 **질문 단계** 설계 후, 수급 가능 여부·예상 지급일수/금액 등 결과 화면 설계.  
- **공통 활용**: 인트로·헤더·푸터·카드·버튼 스타일은 퇴직금과 동일 디자인 시스템 사용.  
- **전환**: 퇴직금 결과/실업급여 결과 화면 등에서 "다른 걸 해보기" 링크로 `service` 변경 후 인트로 또는 해당 Step 1로 이동 가능하도록 구현 가능.

## 4. 요약

- **탭 배치**: 상단에 탭을 먼저 두기보다, **통합 인트로 한 장**을 두고 그 안에 **[퇴직금 확인하기] / [실업급여 확인하기]** 두 개 CTA를 배치하는 구조를 권장.  
- **플랜 전체**: (0) 인트로 + 퇴직금/실업급여 선택 → (1~6) 웜 톤 디자인 적용 순서로 진행하면, "메인 화면 가치 전달 + 친근한 UI"를 동시에 만족시킬 수 있음.  
- **실업급여**: 위 인트로에서 선택 시 실업급여 전용 자격·계산 플로우를 신규 구현하면 되며, 상세 요건은 노동부·고용24 기준을 참고하여 설계하면 된다.
